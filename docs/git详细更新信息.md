# Git 详细更新信息

更新时间：2026-04-10

## 本次更新概述

本次更新主要围绕三个方向展开：

1. 关系网能力增强  
在关系网页面新增基于大模型的自动关系生成能力。系统可以从书籍开头读取到“最后一个已生图章节”为止的内容，并生成当前阅读进度下的人物关系图，同时在界面中标注“这是到 XX 章节为止的角色关系图”。

2. AI 对话系统接入  
在关系网页面右侧新增 AI 对话面板，支持两种模式：
- `AI 伴读`
- `扮演书中角色`

角色扮演模式会读取当前书籍的角色设定、社会关系和阅读进度范围，只依据当前可知内容作答，不再默认知道后续剧情。

3. 本地图片与索引整理  
对 `pic_db/` 中的本地图片命名和索引逻辑做了进一步整理，统一采用更可读的中文命名方式，并修复了若干因旧命名、重命名或文件内容错位导致的前端显示异常。

## 代码层改动

### 1. 关系网新增 AI 生成功能

涉及文件：
- `services/aiService.ts`
- `App.tsx`
- `components/SocialNetwork.tsx`

主要变更：
- 新增基于火山 `doubao-seed-2-0-pro-260215` 的关系分析调用。
- 新增“按阅读进度生成关系图”逻辑。
- 自动计算某本书“最后一个已生图章节”。
- 仅使用当前阅读进度内的正文构建人物关系，不提前泄露后续剧情。
- 生成结果直接替换该书当前关系数据。

界面表现：
- 关系网顶部新增 `AI 生成关系图` 按钮。
- 生成完成后显示“已生成 N 条关系，范围截至 XX 章节”。
- 页面顶部保留“这是到 XX 章节为止的角色关系图”的提示。

### 2. 关系网新增 AI 聊天面板

涉及文件：
- `components/SocialNetwork.tsx`
- `services/aiService.ts`
- `types.ts`
- `services/storageService.ts`
- `App.tsx`

主要变更：
- 在关系网右侧新增 AI 对话面板。
- 支持“AI 伴读”和“扮演书中角色”两种模式。
- 角色模式会读取：
  - 当前书籍角色设定
  - 当前角色相关社会关系
  - 当前阅读进度之前的正文内容
- 伴读模式仍可基于整本书内容做剧情解释与关系梳理。

### 3. 聊天记录持久化

涉及文件：
- `types.ts`
- `services/storageService.ts`
- `App.tsx`
- `components/SocialNetwork.tsx`

主要变更：
- 新增 `ChatMessage` 与 `RelationshipChatState` 数据结构。
- 将聊天记录按“书籍维度”保存到 IndexedDB。
- 刷新页面后，切回对应书籍仍能继续查看之前对话。
- 删除书籍时，会同步清理该书对应的聊天记录。

### 4. 角色模式阅读范围限制

涉及文件：
- `components/SocialNetwork.tsx`
- `services/aiService.ts`

主要变更：
- 聊天提示词新增“角色只能知道当前阅读进度之前剧情”的限制。
- 若用户询问超出当前进度的内容，模型需明确表示暂时不知道。
- 聊天面板头部新增范围提示：
  - `当前角色可知范围：截至第 X 章`

### 5. 聊天头像与清空对话

涉及文件：
- `components/SocialNetwork.tsx`
- `types.ts`

主要变更：
- 回复消息新增头像展示。
- 角色回复使用资产库中对应角色的头像。
- `AI 伴读` 使用机器人头像。
- 历史消息会记录回复者身份，避免切换角色后旧消息头像串位。
- 新增 `清空对话` 按钮，支持清空当前书籍的聊天记录并同步更新本地持久化。

## 图片与本地存储改动

涉及文件：
- `services/localImageService.ts`
- `vite.config.ts`
- `App.tsx`
- `components/AssetLibrary.tsx`
- `pic_db/`

主要变更：
- 本地图片命名规则调整为更可读的中文命名：
  - 资产图：直接使用资产名，如 `孙悟空.jpg`
  - 章节插图：使用 `章节名-第N段.jpg`
  - 封面：统一使用 `封面.jpg`
- 已对 `pic_db/` 中大量旧文件执行重命名迁移。
- 修复《小红帽》等书籍中部分角色图内容与文件名不一致的问题。
- 修复封面、角色图、地点图、插图在刷新后的本地索引恢复逻辑。
