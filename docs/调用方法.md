# 调用方法

本文档汇总当前项目实际使用的火山方舟接口形式。为避免在文档中继续暴露真实密钥，以下示例统一使用环境变量 `ARK_API_KEY`。

## 1. 文生图

当前项目用于：

- 书籍封面生成
- 角色设定图
- 地点设定图
- 段落插图

### 1.1 Seedream 4.5

```bash
curl -X POST https://ark.cn-beijing.volces.com/api/v3/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ARK_API_KEY" \
  -d '{
    "model": "doubao-seedream-4-5-251128",
    "prompt": "儿童故事插画，山坡上放羊的孩子，温暖日光，水彩绘本风格",
    "sequential_image_generation": "disabled",
    "response_format": "url",
    "size": "2K",
    "stream": false,
    "watermark": true
}'
```

  -H "Authorization: Bearer $ARK_API_KEY" \

## 3. 关系图生成与聊天

当前项目中的以下功能使用 `doubao-seed-2-0-pro-260215`：

- 关系页按阅读进度生成角色关系图
- AI 伴读聊天
- 扮演书中角色聊天

### 3.1 Responses API 基础形式

```bash
curl https://ark.cn-beijing.volces.com/api/v3/responses \
  -H "Authorization: Bearer $ARK_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seed-2-0-pro-260215",
    "input": [
      {
        "role": "user",
        "content": [
          {
            "type": "input_text",
            "text": "请根据以下内容总结人物关系。"
          }
        ]
      }
    ]
}'
```

### 3.2 图文输入形式

```bash
curl https://ark.cn-beijing.volces.com/api/v3/responses \
  -H "Authorization: Bearer $ARK_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seed-2-0-pro-260215",
    "input": [
      {
        "role": "user",
        "content": [
          {
            "type": "input_image",
            "image_url": "https://ark-project.tos-cn-beijing.volces.com/doc_image/ark_demo_img_1.png"
          },
          {
            "type": "input_text",
            "text": "你看见了什么？"
          }
        ]
      }
    ]
}'
```

## 4. 项目内模型分工

| 功能 | 当前模型 |
|---|---|
| 章节扫描角色/地点 | `deepseek-v3-2-251201` |
| 段落叙事分析 | `deepseek-v3-2-251201` |
| 关系页关系图生成 | `doubao-seed-2-0-pro-260215` |
| AI 伴读与角色对话 | `doubao-seed-2-0-pro-260215` |

## 5. 当前项目内的调用路径

### 5.1 文本与关系

- `services/aiService.ts`
  - `scanChapterForAssets()`
  - `analyzeNarrative()`
  - `analyzeRelationships()`
  - `analyzeRelationshipsFromReadingProgress()`
  - `chatWithBookRole()`

### 5.2 图片生成

- `generateIllustration()`
- `generateAssetVisual()`
- `generateBookCover()`

### 5.3 本地保存

图片生成后不会只保留远程 URL，而是会进一步经过：

- `services/localImageService.ts`
- `vite.config.ts` 中的本地中间件

最终落盘到：

- `pic_db/<书名>/covers/books/`
- `pic_db/<书名>/assets/characters/`
- `pic_db/<书名>/assets/locations/`
- `pic_db/<书名>/illustrations/paragraphs/`
