# 智绘阅读软件简介

版本：2026-04-10  
文档属性：项目提交版

## 1. 软件定位

智绘阅读是一款面向小说、寓言、神话与科普文本的 AI 阅读增强工具。系统围绕“读文本、抽取世界观、生成统一风格插图、保存图片资产、在关系页继续阅读理解与对话”这一链路构建，目标是把传统纯文本阅读升级为可持续沉淀视觉资产的图文阅读体验。

它不是单独的 AI 绘画工具，也不是传统电子书阅读器，而是一套把 `阅读器`、`世界观库`、`关系图`、`AI 伴读` 和 `本地图片存档` 整合到一起的阅读工作台。

图1 总体架构图

## 2. 当前版本核心能力

- 导入 `.txt` 文本并生成书籍
- 上传封面或使用 AI 生成封面
- 阅读器中逐段触发生图、重生成与删除
- 对章节进行角色、地点与关系扫描
- 自动建立角色与地点设定图
- 批量生图，支持按顺序分批并发
- 世界观页中管理角色与地点资产
- 关系页中一键生成“截至当前阅读进度”的关系图
- 关系页中与 AI 伴读或书中角色聊天
- 将图片自动保存到本地 `pic_db/`
- 将业务状态和聊天记录保存到 IndexedDB

## 3. 当前界面组成

### 3.1 书架页

书架页负责导入书籍、选择书籍、管理封面和删除书籍。

![书架页](./assets/ui-bookshelf.png)
图2 书架页

### 3.2 阅读器页

阅读器页承担正文阅读、单段生图、批量生图、扫描设定、切换模型与导出等核心流程。

![阅读器页](./assets/ui-reader.png)
图3 阅读器页

### 3.3 世界观页

世界观页用于集中查看角色与地点资产，并追踪每张设定图的生成状态。

![世界观页](./assets/ui-assets.png)
图4 世界观页

### 3.4 关系页

关系页不仅用于查看人物关系图，还可以：

- 用 LLM 阅读到当前已生图章节并生成关系图
- 与 AI 伴读对话
- 选择扮演书中角色对话
- 让角色只基于当前阅读进度之前的剧情回答

## 5. 技术与运行方式

当前版本采用：

- React 19 + TypeScript + Vite 6
- 火山方舟模型接口
- IndexedDB 本地状态存储
- 仓库本地 `pic_db/` 图片存储

图片和状态双持久化后，系统不再只依赖远程临时 URL，能够在本地长期保留历史生成结果。

![运行时数据流图](./assets/runtime-dataflow.svg)
图15 运行时数据流图

## 6. 当前版本特点

相较于早期版本，当前软件已经有以下明显变化：

- 从“单次段落生图”演进为“阅读工作流工具”
- 从“只看图片 URL”演进为“本地图片库 + 数据库存档”
- 从“手工关系管理”演进为“按阅读进度自动生成关系图”
- 从“只会生图”演进为“还能在关系页继续伴读与角色扮演”

## 7. 结语

智绘阅读当前已经具备较完整的本地原型能力，能够支撑文本导入、阅读、生图、世界观沉淀、关系生成、角色对话与图文导出等一整套多模态阅读流程。它既可以作为 AI 阅读产品原型，也可以作为面向课程、展示和答辩的综合型软件作品。
